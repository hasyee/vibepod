## Variable Naming Rules

Never use abbreviated variable names. Always use full descriptive names:

- for example use `result` instead of `r`
- `i`/`j` are acceptable for indexes
- `a`/`b` are acceptable in sort callback parameters

## Formatting Rules

When writing or editing code, always apply formatting in this priority order:

1. If a `.prettierrc` / `.prettierrc.json` / `.prettierrc.js` etc. exists in the project root → use those rules
2. Otherwise → follow the user's global VSCode settings
